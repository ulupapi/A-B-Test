# Стратегия A/B тестирования: логика расчета

Этот проект показывает, как последовательно решается матричная игра с нулевой суммой для выбора стратегии рекомендаций.

## Исходные данные

- Игрок A (команда) максимизирует прирост конверсии.
- Игрок B (внешняя среда) минимизирует прирост конверсии игрока A.
- Матрица выигрышей A:
  - A1: [9, 17, 7]
  - A2: [15, 7, 13]

## Логика решения

1. Проверка доминирования  
Сравниваются стратегии игроков поэлементно.  
Для B столбец B1 доминируется столбцом B3, потому что:
- 7 < 9
- 13 < 15  
Значит, B1 исключается как нерациональный выбор для минимизатора.

2. Проверка седловой точки в исходной игре  
Считаются:
- нижняя цена игры `maxmin`
- верхняя цена игры `minmax`  
Для исходной матрицы получается:
- `maxmin = 7`
- `minmax = 13`  
Так как значения не равны, седловой точки нет, и нужны смешанные стратегии.

3. Решение сокращенной игры 2x2  
После удаления B1 остается матрица:
- A1: [17, 7]
- A2: [7, 13]

Ожидаемые выигрыши A как функции `p1 = P(A1)`:
- `g1(p1) = 17p1 + 7(1-p1)`
- `g2(p1) = 7p1 + 13(1-p1)`  

В оптимуме выполняется равенство `g1(p1*) = g2(p1*)`, откуда:
- `p1* = 0.375`
- `p2* = 0.625`
- цена игры `V = 10.75`

4. Восстановление смешанной стратегии B для исходной 2x3  
Так как B1 доминируется и исключается, ему соответствует нулевая вероятность:
- `q1 = 0`
- `q2 = 0.375`
- `q3 = 0.625`

5. Контроль через альтернативные методы  
- ЛП-формулировка (симплекс) дает ту же цену игры `V = 10.75`.
- Метод Брауна–Робинсона дает приближение, которое сходится к `V` при увеличении числа итераций.

## Что показывает приложение

- этапы решения (доминирование, maxmin/minmax, смешанные стратегии);
- график графоаналитического метода;
- сходимость метода Брауна–Робинсона;
- итоговое сравнение результатов разных подходов.
