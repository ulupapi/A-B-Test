# Стратегия A/B тестирования алгоритма рекомендаций (Теория игр)

Приложение реализует решение нулевой суммы для матрицы выигрышей Игрока A:

- `A1`: `[9, 17, 7]`
- `A2`: `[15, 7, 13]`

## Ключевые результаты

1. Доминирование:
- Для Игрока A доминируемых строк нет.
- Для Игрока B столбец `B1` доминируется `B3` (так как `7 < 9` и `13 < 15`), поэтому `B1` исключается.

2. Седловая точка в исходной `2x3`:
- `maxmin = 7`
- `minmax = 13`
- Седловой точки нет, нужны смешанные стратегии.

3. После редукции к `2x2` (`B2`, `B3`):
- Матрица: `[[17, 7], [7, 13]]`
- Оптимальная стратегия A: `p1 = 3/8 = 0.375`, `p2 = 5/8 = 0.625`
- Оптимальная стратегия B в исходной `2x3`: `q1 = 0`, `q2 = 3/8 = 0.375`, `q3 = 5/8 = 0.625`
- Цена игры: `V = 43/4 = 10.75`

4. Методы в приложении:
- Графоаналитический метод (график прямых `g_j(p1)` и нижней огибающей)
- Симплекс-метод через `scipy.optimize.linprog`
- Метод Брауна–Робинсона (100+ итераций, таблица и график сходимости)
